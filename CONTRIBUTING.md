# Contributing

Thanks for your interest in contributing.

## Development Setup

```bash
pnpm install
pnpm dev
```

## Before Opening a PR

- run `pnpm build`
- run `pnpm lint`
- verify upload/download and PIN flow manually
- keep changes focused and small

## Branch and Commit Style

- use short feature branches
- follow Conventional Commits
- include context in commit body when changes are non-trivial

## Pull Request Checklist

- clear problem statement
- summary of the implementation
- screenshots/video for UI changes (if applicable)
- notes about risks and follow-up work
